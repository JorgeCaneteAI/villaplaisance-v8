### Pièges MariaDB → MySQL rencontrés (à retenir)

Le dump `database/vp_v7.sql` est généré sur **MariaDB** (sur o2switch) et
contient deux incompatibilités avec MySQL 8 :

1. **Collation `utf8mb4_uca1400_ai_ci`** (spécifique MariaDB 10.10+, inconnue
   de MySQL). Solution : créer un dump dérivé avec `utf8mb4_unicode_ci`
   (compatible des deux côtés) :
   ```bash
   sed 's/utf8mb4_uca1400_ai_ci/utf8mb4_unicode_ci/g' \
     database/vp_v7.sql > database/vp_v7-mysql.sql
   ```

2. **Foreign keys avec noms en doublon** (`CONSTRAINT \`1\``). MariaDB
   tolère des FK avec le même nom dans deux tables différentes du même
   schéma, MySQL refuse. Solution : renommer la 2e occurrence
   (`fk_itinerary_steps_itinerary`).

Le dump `vp_v7-mysql.sql` est gitignoré (généré à la demande, pas de
duplication dans le repo).

### Lancement local — commande définitive

Le serveur PHP built-in route TOUTES les requêtes vers `index.php` quand
on lui passe ce fichier en router. Du coup les fichiers statiques
(CSS, JS, images) ne sont jamais servis. Solution : un `router.php` à
la racine qui retourne `false` pour les fichiers existants et délègue
le reste à `public/index.php`. Lancer avec `-t public` (DocRoot =
`public/`) pour que `return false` serve le fichier depuis le bon dossier.

```bash
cd ~/Documents/C.L.A.U.D.E/villaplaisance-v8
php -S localhost:8767 -t public router.php
```

DB :
```bash
MYSQL=/Applications/MAMP/Library/bin/mysql80/bin/mysql
$MYSQL -u root -proot -h 127.0.0.1 -P 8889 villaplaisance_v8_dev
```

### Piège rencontré : `.htaccess` écrasé par le `.cpanel.yml`

Le `.cpanel.yml` initial faisait `cp .htaccess $DEPLOYPATH/public/.htaccess`,
ce qui écrasait le bon `public/.htaccess` (1150 bytes, contient PWA + Force
HTTPS + routing) avec le mini `.htaccess` racine (125 bytes, juste un redirect
interne vers `/public/`).

Le mini `.htaccess` racine sert uniquement quand le DocumentRoot Apache est à
la racine du projet ; ici le sous-domaine pointe directement sur `public/`,
donc on ne le copie pas. Ligne supprimée du `.cpanel.yml`.
